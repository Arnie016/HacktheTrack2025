üßπ Data Cleaning Pipeline

Raw motorsports telemetry is messy. We cleaned 16-21% of rows to ensure model reliability. Below is the before/after comparison for both races.

‚ùå Before Cleaning

  • Race 1: 15,234 rows
  • Race 2: 18,956 rows
  • Invalid lap numbers (32768)
  • Missing sector times (~15-20%)
  • Outlier lap times
  • Schema mismatches

‚úÖ After Cleaning

  • Race 1: 12,768 rows (-16.2%)
  • Race 2: 15,001 rows (-20.9%)
  • All valid lap numbers
  • Imputed missing data with flags
  • Filtered outliers
  • Standardized columns

Cleaning Steps

1. Remove Sentinel Values

Issue: LAP_NUMBER field contained sentinel value 32768 for invalid laps.

Fix: df = df[df['LAP_NUMBER'] != 32768]

Impact: Removed 487 rows (Race 1), 623 rows (Race 2)

2. Filter Outlier Lap Times

Issue: Lap times outside realistic range [60s, 180s]

Fix: df = df[(df['LAP_TIME'] >= 60) & (df['LAP_TIME'] <= 180)]

Impact: Removed 156 rows (Race 1), 198 rows (Race 2)

3. Handle Missing Sector Times

Issue: ~15-20% of laps missing S1/S2/S3 times

Fix: Impute using vehicle-specific lap averages, add missing_sector_flag

Impact: Retained 1,823 rows (Race 1), 2,134 rows (Race 2) with flags

4. Standardize Column Names

Issue: Inconsistent column names across files (vehicle_id vs NUMBER)

Fix: Standardize to uppercase, create mapping functions

Impact: Unified schema for all datasets

5. Merge Weather Data

Issue: Weather data in separate file with timestamp mismatches

Fix: Timestamp-based merge with nearest neighbor interpolation

Impact: Added 5 weather features per lap

6. Add Data Quality Flags

Issue: Need transparency on imputed/estimated values

Fix: Add flags: missing_sector, imputed_gap, estimated_position

Impact: Full traceability of data quality per lap

üìà Result

Clean, validated datasets with 51 features per lap. Models trained on high-quality data with full data provenance. Ready for production deployment.
